# 信息整理系统 PRD

## 1. 产品目标

将零散课程通知转换为可确认、可管理的任务。产品先完成个人任务管理，再扩展为班级协作网页，由课代表或班委发布任务并同步给班级成员，最终发展为轻量化 App。

## 2. 用户与权限

- **平台所有者**：系统最高权限，仅限产品负责人；可以查看和管理所有班级、用户及任务。
- **班级创建者**：管理所在班级、成员和管理员。
- **管理员**：发布班级任务。全班撤回功能暂缓。
- **普通成员**：接收班级任务，维护自己的完成状态和个人任务。

当前每位用户只能加入一个班级。

## 3. 核心功能

### 3.1 课程任务

- Input 页面支持结构化填写和 AI 整段识别。
- Confirm 页面用于检查解析结果。
- Board 页面按 `Done → Today → Upcoming` 管理任务。
- 每条作业或项目生成一张独立任务卡片。
- 使用线条粗细表达重要程度、聚焦和逾期状态。

### 3.2 个人任务回收站

- 任务卡片右上角提供删除按钮。
- 删除前必须二次确认，确认后移入回收站。
- 支持恢复、单个永久删除和一键清空回收站。
- 当前只处理个人任务；班级任务全班撤回暂不实现。

### 3.3 登录与班级协作

- 后台预先录入姓名、学号、所属班级和角色。
- 用户使用学号和服务器配置的初始密码登录。
- 首次登录必须修改密码。
- 管理员发布班级任务后，成员自动收到任务，但每位成员的完成状态相互独立。

## 4. 试运行计划

1. 完善个人任务和回收站。
2. 在开发电脑上运行本地后端和数据库。
3. 使用个人热点连接少量测试手机，完成网页端试运行。
4. 增加登录、权限、班级任务和同步功能。
5. 产品成熟后再研究公网部署、PWA、App、品牌和上线方案。

## 5. 当前范围

- 本地试运行阶段不公开到公网。
- 暂不支持一人加入多个班级。
- 暂不实现管理员全班撤回任务。
- 暂不实现图片识别、多设备正式部署和应用商店发布。
- 当前个人数据使用 `localStorage`；引入后端后迁移到服务器数据库。
